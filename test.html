GitHub 페이지에서 HTML 확장자 제거

GitHub 페이지를 사용하여 최신 사이트를 호스팅하고 있습니다 ( http://jasonhoffmann.github.io/ ). 하위 페이지 끝에서 .html 확장자를 제거 할 수있는 방법이 있는지 궁금합니다 ( http://jasonhoffmann.github.io/contact.html ) 섬기는 사람.

2014. 1. 20.Jason Hoffmann

하위 디렉토리를 사용해야한다고 가정하지만 더 간단한 방법이있을 수 있습니다.

/index.html            => http://jasonhoffmann.github.io
    /contact/index.html    => http://jasonhoffmann.github.io/contact

그러나 CNAME을 사용하여 GitHub 페이지를 자신의 도메인 으로 리디렉션하도록 제어 할 수 있습니다. 죄송 합니다만, 귀하를위한 명확한 답변은 없지만, 이것이 귀하를 시작할 수 있기를 바랍니다.

14
2014. 1. 20.Sam

그것은 이미 작동하고 아무것도 할 필요가 없습니다

이미 알고 있는지 확실하지 않지만 GitHub 페이지에서 .html 확장자를 제거하려면 링크에서 .html 확장자를 제거하면됩니다.

즉, 이미 작동하고 아무것도 할 필요가 없습니다. 하위 디렉토리, CNAME, 리디렉션, Jekyll _config.yml, YAML 최전방 또는 여기의 모든 답변에서 제안 된 기타 해킹을 사용할 필요가 없습니다.

예

예를 들어 다음을 사용할 수 있습니다.

  • https://rsp.github.io/gh-pages-no-extension/test1

대신에:

  • https://rsp.github.io/gh-pages-no-extension/test1.html

동일한 파일을 표시합니다. 링크에서 링크를 변경하면됩니다.

테스트

링크를 클릭 할 수있는 위치에서 실제로 어떻게 작동하는지 보여주기 위해 몇 가지 테스트를 수행했으며 특정 경우 안전하지 않은 리디렉션에 대한 경고를 포함하여 어떤 파일이 어떤 URL로로드되는지 강조 표시합니다.

Eaxample의 경우 링크 :

  • https://rsp.github.io/gh-pages-no-extension/test1

uRL에서 test1로 표시되지만 실제로 test1.html 파일을 표시합니다.

다른 솔루션의 문제

다른 답변에서 여기에 제안 된대로 특수 하위 디렉토리에서 index.html과 같은 것을 사용하면 원하는 것을 수행하지 않을 것입니다. example.com/name.html를 요청할 때 단순히 example.com/name를 제공하지만 대신 301을 제공합니다 example.com/name/ 파일로 리디렉션하면 example.com/name/index.html 파일의 내용이 표시됩니다.

그 결과 다음과 같은 문제가 발생합니다. 다른 디렉토리에 있고 동일한 레벨의 다른 페이지에 대한 링크에 <a href="../name"> 대신 <a href="name">를 사용해야하며 모든 자산에 대해 동일한 이동 ( 또는 누군가가 제안한대로 절대 경로를 사용할 수 있습니다 .GitHub 페이지, 특히 프로젝트 페이지에는 좋지 않습니다. 포크 및 프로젝트 이름을 바꾼 후에는 모든 HTML 파일에서 모든 링크 및 js/css 자산을 업데이트해야합니다. ).

다른 것은 이제 모든 탐색에 쓸모없는 리디렉션을 가지고 있다는 것입니다. 이는 우연히 HTTPS에서 HTTP로 리디렉션합니다 ( 슬래시가없는 secure URL 에서 슬래시가있는 안전하지 않은 URL) 보다:

  • https://rsp.github.io/gh-pages-no-extension/dir (보안 HTTPS)

(작성 당시)는 다음으로 리디렉션됩니다.

  • http://rsp.github.io/gh-pages-no-extension/dir/ (안전하지 않은 HTTP)

그리고 [~ # ~] not [~ # ~] :

  • https://rsp.github.io/gh-pages-no-extension/dir/ (보안 HTTPS)

예상 한대로이를 알고 있어야합니다. 보다: (이 이미지는 보안 HTTPS URL에 연결되어 있지만 안전하지 않은 HTTP 연결을 사용하게됩니다- 시크릿 모드를 사용하여 .)

2016 년 6 월 17 일 현재 헤더 모양은 다음과 같습니다.

$ curl -I https://rsp.github.io/gh-pages-no-extension/dir
    HTTP/1.1 301 Moved Permanently
    Server: GitHub.com
    Content-Type: text/html
    Location: http://rsp.github.io/gh-pages-no-extension/dir/
    ...       ^^^^

잘만되면 GitHub가 앞으로이를 고칠 것입니다. (거의 1 년 전에이 답변을 작성할 때 발견했으며이 문제는 2016 년 6 월 현재 존재합니다.)

이상하게도 2016 년 6 월 17 일 현재 (이 답변을 업데이트하는 동안 최근에 발견했기 때문에 항상 그런지 확실하지 않습니다)이 URL :

  • https://rsp.github.io/gh-pages-no-extension (보안 HTTPS)

다음으로 리디렉션 :

  • https://rsp.github.io/gh-pages-no-extension/ (보안 HTTPS)

그것은해야합니다. 헤더를 참조하십시오 :

$ curl -I https://rsp.github.io/gh-pages-no-extension
    HTTP/1.1 301 Moved Permanently
    Server: GitHub.com
    Content-Type: text/html
    Location: https://rsp.github.io/gh-pages-no-extension/
    ...       ^^^^^

유일한 차이점은 그것이 프로젝트 루트 URL이고 프로젝트 내부의 더 깊은 디렉토리가 아니라는 것입니다 ( 프로젝트 구조 참조) 버그가있는 리디렉션은 프로젝트의 GitHub 페이지 내부의 딥 링크에만 존재하는 것으로 보입니다 시크릿 모드를 사용할 때만 한 브라우저에서 테스트했습니다.

결론은 조심해야한다는 것입니다 . 내 조언은 모든 리디렉션을 모두 피하고 리디렉션하지 않는 URL 만 사용하는 것입니다.

GitHub Pages에 내장 된 정적 사이트 생성기 인 Jekyll을 통해이를 수행 할 수 있습니다. 여기에는 _config.yml (사이트의 모든 페이지 및 게시물에 영향을 미침) 또는 YAML 최전방 에서 설정할 수있는 퍼머 링크 설정 = 각 페이지/포스트 당.

예를 들어, 다음 코드를 contact.html 파일에 추가 할 수 있습니다.

---
    permalink: contact/
    ---

이것은 <!DOCTYPE html> 앞에 삽입되지만 Jekyll은 GitHub 페이지에서 제공 될 때이를 제거하도록주의를 기울입니다. 이렇게하면 jasonhoffman.github.io/contact 대신 jasonhoffman.github.io/contact.html의 페이지에 액세스 할 수 있습니다. Jekyll이 실제로하고있는 일은 contact라는 디렉토리를 만들고 그 안에 index.html 내용으로 contact.html 파일을 넣는 것입니다. Jekyl을 로컬로 설치하고 실행 인 경우 다음과 같이 표시됩니다.

.
    |- _config.yml (optional)
    |- contact.html
    |- index.html
    |- css
       |- styles.css
    |- img
       |- image.jpg
    |- _site
       |- index.html
       |- contact
          |- index.html
       |- css
          |- styles.css
       |- img
          |- image.jpg

Jekyll을 로컬로 설치하지 않으면 _site 폴더를 제외한 모든 것을 볼 수 있습니다. 이것을 GitHub로 푸시하면 서버가 Jekyll을 실행하고 _site 폴더와 동일한 폴더를 생성하여 제공합니다 (GitHub 리포지토리에는 표시되지 않음).

Jekyll을 건너 뛰고 내부에 index.html가있는 contact 서브 디렉토리를 작성할 수도 있습니다. 어느 쪽이든 사이트의 자산 (img, css, js)에 대한 링크가 앞에 /인지 확인해야합니다. 예를 들어

<link rel="stylesheet" href="css/style.css">

대신에

<link rel="stylesheet" href="/css/style.css">

북극 여우 테마 를 사용하고 있는데 다음 링크 문자열을 사용합니다 :

<a class="page-link" href="{{ page.url | prepend: site.baseurl }}">{{ page.title }}</a>

해결책은 remove: '.html'를 필터로 사용하면

<a class="page-link" href="{{ page.url | remove: '.html' | prepend: site.baseurl }}">{{ page.title }}</a>

Jekyll을 통한 로컬 서비스 현재 작동하지 않지만 Jekyll 3.0에 대해 약속되어 있습니다 : https://github.com/jekyll/jekyll/pull/3452
